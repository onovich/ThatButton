项目交接文档：《核心熔毁：那个键》(Core Meltdown: THAT Button)

1. 项目概况

项目名称：核心熔毁：那个键 (Core Meltdown: THAT Button)

项目定位：快节奏逻辑推理与反应测试类 HTML5 网页游戏。

设计核心：以“别按那个键”为命题，通过反直觉的“扫雷式推理+高压点击”玩法，为玩家提供心流体验。

项目愿景：打造一款零外部依赖、即开即玩、高复玩性的网页端微型游戏。

2. 需求设计文档 (PRD)

目标用户：追求短时间高强度脑力挑战、快节奏点击体验的玩家。

核心玩法：

反向推理：系统实时给出致命按键的逻辑特征，玩家需在倒计时内识别出所有致命按键，并点击剩余的全部安全按键。

难度系统：基于层级(Level)的动态难度调整，逻辑复杂度（AND/OR/NOT组合）随关卡推进自动提升，倒计时压力动态收紧。

胜负判定：按下致命键或超时则判定为系统失败；完成当前面板所有安全按键的点击则进入下一关。

演示优势：单屏交互，全触控/鼠标适配，极高的视听反馈密度。

3. 技术文档

技术栈：原生 HTML5 + CSS3 (Tailwind) + Vanilla JavaScript。

关键实现：

UI/渲染：采用 CSS Grid 布局构建 3x3 交互阵列；通过 CSS 动画实现 CRT 扫线效果、屏幕震动及故障滤镜。

逻辑引擎：基于 generateRule 函数的闭包逻辑，通过布尔运算动态生成符合特征的“致命键”集合。

音频系统：基于 Web Audio API 实现原生音频合成，无外部音频资产依赖，确保运行稳定性与加载速度。

交互适配：配置 touch-action: manipulation 优化移动端点击延迟。

4. 架构设计规划

模块划分：

gameState: 全局状态控制（时间、分数、当前规则、按键属性集合）。

generator: 规则生成器，负责根据关卡难度动态匹配布尔逻辑。

audioEngine: 原生音频合成模块，负责交互反馈音与告警音。

gameLoop: 核心帧循环，驱动倒计时更新、渲染逻辑及胜负判定。

扩展性说明：

逻辑规则模板化存储在 templates 数组中，支持通过配置式添加新的推理规则，无需重构底层引擎。

5. 开发路线图 (Roadmap)

阶段一 (当前状态)：核心逻辑闭环，完成基础规则生成、点击反馈及游戏循环。

阶段二 (美术进阶)：

强化视觉表现，增加 CRT 屏幕曲面畸变滤镜。

引入粒子反馈效果，提升点击打击感。

阶段三 (内容扩充)：

增加干扰逻辑（如屏幕晃动、线索暂时模糊等“恶劣环境”机制）。

接入本地存储 (LocalStorage) 以实现得分排行榜功能。

阶段四 (项目发布)：

代码压缩与跨浏览器兼容性测试。

准备上线前的优化部署，确保无外链依赖的前提下平稳运行。

6. 注意事项

资源合规：所有视觉和音频资源必须保持原生渲染，以保证加载性能。

性能优化：在复杂逻辑更新时，需关注 requestAnimationFrame 的执行效率，避免因DOM重绘过频导致的操作迟滞。
